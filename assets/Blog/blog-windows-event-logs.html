    Windows Event Logs and System Monitor Research

    Research May 2, 2026 Nguyễn Hữu Duy Nhất

    I. Khái niệm Windows Event Logs

    1.1. Khái niệm Windows Event Logs

    Windows Event Logs đóng vai trò là một nguồn chứng cứ điện tử cốt lõi. Hệ thống này ghi nhận toàn diện các biến động liên quan đến software, hardware, OS, security. Service Windows Event Log chịu trách nhiệm quản lý các sự kiện, nhật ký sự kiện,… nó thu thập các sự kiện từ nhiều nguồn khác nhau và lưu trữ tập trung tại một thư mục. Các sự kiện này giúp cho ta biết được các thao tác đã xảy ra trên hệ thống, những sự kiện xảy ra vào thời gian nào, tham chiếu đến user cụ thể liên quan tới sự kiện hay hành vi nào đó hoặc là các thông tin khác trong hệ thông.

    1.2. Windows Event Logs được tọa lạc ở chộ nào?

    Hiện nay, với các bản Windows mới thì Winlog được lưu trữ tại: %systemroot%\System32\winevt\logs
    Event log có định dạng XML và phần mở rộng là .evtx.

    1.3. Format Log

    Trước đây, từ phiên bản Win 2008 trở về trước thì các tệp log được ánh xạ trực tiếp và bộ nhớ, thì điều này là 1 vấn đề lớn trong việc quản lý bộ nhớ vì bộ nhớ luôn phải dành 1 không gian tầm 300MB chỉ để cho việc cấp phát các event log và tất nhiên hiệu suất sẽ bị giảm nếu dung lượng của bộ nhớ không lớn. Các user hồi xưa sẽ thường hay tắt logging đi để nâng cao hiệu suất bộ nhớ. Nhưng trong những bản Window sau này thì việc điều này đã được khắc phục, service Windows Event Log quản lý các event và event logs (logging events, querying events, subscribing to events, archiving event logs, and managing event metadata). Trong bộ nhớ lúc này chỉ chứa những header nhỏ với các đoạn mã 64KB nằm trong bộ nhớ và nó thực sự đã giải quyết được vấn đề quản lý bộ nhớ mà không làm giảm hiệu suất.

    Định dạng của event log cũng thay đổi với extension là .EVTX và được lưu trữ ở định dạng XML, giúp việc giải mã cấu trúc nhật ký trở nên trực quan hơn và cũng dễ dàng để tạo bộ lọc hỗ trợ việc tìm kiếm.

    1.4. Event Types

    Có tổng cộng là 5 loại event khác nhau có thể được ghi lại, mỗi sự kiện đều phải thuộc 1 trong 5 loại này và mỗi loại sẽ có 1 biểu tượng khác nhau.

    Event type Description
    Error An event that indicates a significant problem such as loss of data or loss of functionality. For example, if a service fails to load during startup, an Error event is logged.
    Warning An event that is not necessarily significant, but may indicate a possible future problem. For example, when disk space is low, a Warning event is logged.
    Information An event that describes the successful operation of an application, driver, or service.
    Success Audit An event that records an audited security access attempt that is successful.
    Failure Audit An event that records an audited security access attempt that fails.

    1.5. Các loại Log

    Window Event Logs có 3 loại tiêu chuẩn là Security, System, Application. Ngoài ra thì Winlog còn có các bản additional logs đã được thêm vào log chuyên biệt (còn gọi là Custom Log), Ex: Powershell, Task Scheduler,…

    A. Log tiêu chuẩn

    • Security: Lưu trữ các sự kiện liên quan đến Security và Audit của hệ thống. Ghi lại các hoạt động như: đăng nhập thành công/thất bại, thay đổi quyền hạn, truy cập tệp tin, v.v.
    • System: Ghi lại các sự kiện sinh ra bởi hệ điều hành và các thành phần cốt lõi (driver, Windows Services). Rất quan trọng để khắc phục sự cố màn hình xanh (BSOD).
    • Application: Ghi lại các sự kiện sinh ra bởi các phần mềm và ứng dụng của bên thứ ba.

    B. Nhóm log chuyên biệt (Custom log,…)

    Cung cấp độ hiển thị (Visibility) cực kỳ sâu cho các ứng dụng cụ thể như PowerShell, Task Scheduler, và Windows Defender/Sysmon. Đây là chìa khóa để săn tìm mối đe dọa (Threat Hunting).

    II. Security logs và các ví dụ điển hình

    2.1. Security Logs

    Security log mang tính bảo mật khắt khe: chỉ Admin mới có quyền truy cập và chỉ có tiến trình LSASS mới được phép ghi lại các sự kiện bảo mật.

    2.2. Các ví dụ điển hình

    2.2.1. Tấn công Brute-Force Password

    Dấu hiệu nhận diện trọng yếu chính là sự xuất hiện đột biến và dày đặc của Event ID 4625 (An account failed to log on) trong một khoảng thời gian ngắn.

    2.2.2. Tracking Account Usage

    • Event ID 4624: Successful Logon.
    • Event ID 4634 / 4647: Successful Logoff.
    • Event ID 4672: Special Privileges Assigned.

    Trường Logon Type cung cấp ngữ cảnh quan trọng:

    • Type 2 (Interactive): Đăng nhập trực tiếp.
    • Type 3 (Network): Đăng nhập qua mạng (SMB share).
    • Type 10 (RemoteInteractive): Đăng nhập từ xa (RDP).

    III. Sơ lược về System Monitor (Sysmon)

    3.1 Sysmon Event ID

    • Event ID 1 (Process Creation): Ghi nhận tiến trình mới kèm CommandLine và ProcessGUID.
    • Event ID 8 (CreateRemoteThread): Dấu hiệu của Process Injection.
    • Event ID 11 (FileCreate): Ghi nhận khi một tệp được tạo hoặc ghi đè.
    • Event ID 3 (Network Connection): Lưu vết kết nối TCP/UDP.

    3.2 Ưu điểm vượt trội

    Sysmon cho phép lọc dữ liệu linh hoạt qua XML Schema, giúp giảm nhiễu và tập trung vào các hành vi đáng ngờ.

    IV. Mô phỏng lại 1 cuộc tấn công (PrintNightmare CVE-2021-34527)

    • Security Log (ID 4720/4732): Phát hiện hệ quả (tài khoản hacked được tạo và cấp quyền Admin).
    • Sysmon (ID 11/1): Phát hiện hành vi (dịch vụ spoolsv.exe tạo file DLL lạ và sinh ra tiến trình con thực thi mã độc).

    V. Kết luận

    Windows Event Logs và Sysmon là hai mảnh ghép bổ trợ hoàn hảo. Việc kết hợp cả hai nguồn dữ liệu giúp chuyên gia bảo mật không chỉ biết "ai đã vào nhà" mà còn biết chính xác "họ đã làm gì bên trong".
